# 高考招生报名系统设计方案

## [S1] 项目概述

高考招生报名系统，包含报名管理、数据爬取、院校信息、金融风格数据分析、AI 分析五大模块。面向招办老师和考生/家长双端用户。

**项目位置：** `D:\gaokao-system`

## [S2] 技术架构

```
┌─────────────────────────────────────────────────┐
│                  Frontend (React)                │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ 报名系统  │ │ 数据分析  │ │ AI 分析面板     │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
└─────────────────────┬───────────────────────────┘
                      │ REST API
┌─────────────────────┴───────────────────────────┐
│                  Backend (FastAPI)                │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ 用户/报名 │ │ 院校数据  │ │ 分析/AI 接口    │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────┴───────────────────────────┐
│                  数据层                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ SQLite   │ │ Scrapling│ │ 爬虫数据存储     │ │
│  │ 数据库   │ │ 爬虫     │ │ (JSON/CSV)       │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
└─────────────────────────────────────────────────┘
```

**技术选型：**
- 后端：FastAPI + SQLAlchemy + SQLite（先本地，后可换 PostgreSQL）
- 前端：React + Next.js + Tailwind CSS
- 可视化：Apache ECharts（K 线、热力图、星空图）
- 爬虫：Scrapling (StealthyFetcher)
- AI：本地规则引擎 + DeepSeek API（可切换）

## [S3] 数据分析系统

**设计原则：** 用户选择指标 → 系统渲染对应图表，插件式架构。

**可选指标分类：**

| 分类 | 指标 | 图表类型 | 说明 |
|------|------|----------|------|
| 分数线趋势 | 历年分数线 | 折线图/K 线 | 某校某专业 N 年分数线变化 |
| 竞争程度 | 录取率/报录比 | 柱状图 | 招生数 vs 报名数 |
| 地理分布 | 院校地域分布 | 热力图/星空图 | 按省份看院校密度和竞争 |
| 专业对比 | 各专业热度 | 雷达图/桑基图 | 报名人数流向哪些专业 |
| 个人匹配 | 分数匹配度 | 散点图/气泡图 | 你的分数 vs 目标院校 |
| 招生计划 | 招生人数变化 | 面积图 | 各校扩招/缩招趋势 |

**插件扩展机制：**
- 每个指标是一个 Python 文件，放在 `analysers/` 目录
- 实现标准接口：`get_data(params)` + `render(data)`
- 系统自动发现并注册到指标列表
- 新增指标无需改核心代码或前端

## [S4] 爬虫系统

**插件式架构，按需扩展数据源。**

```
spiders/
├── registry.py          # 爬虫注册表
├── base.py              # 基类 SpiderBase
├── sites/
│   ├── yangguang.py     # 阳光高考网
│   ├── gaokao_com.py    # 掌上高考
│   ├── gaokao_net.py    # 高考网
│   └── custom/          # 用户自定义爬虫
```

**每个爬虫插件实现标准接口：**
- `crawl(params)` — 爬取逻辑
- `parse(raw_data)` — 解析为标准格式

**数据源优先级：**

| 优先级 | 平台 | 数据 | 难度 |
|--------|------|------|------|
| Tier 1 | 阳光高考网 | 院校库、分数线、招生章程、专业库 | 低 |
| Tier 1 | 掌上高考 | 院校详情、历年录取分数、分数线图表 | 低 |
| Tier 2 | 高考网 | 大学排名、一分一段表 | 中 |
| Tier 3 | 各高校官网 | 就业质量报告、招生章程 PDF | 中高 |

## [S5] AI 分析系统

**设计原则：** AI 可选、可切换、不强制。

**双模式：**

| 模式 | 说明 | 成本 |
|------|------|------|
| 本地模式 | 规则引擎 + 简单统计分析 | 免费 |
| 云端模式 | 调用 DeepSeek/其他 LLM | 按量付费 |

**招考分析 Skill 功能：**

| 功能 | 本地 | 云端 | 说明 |
|------|------|------|------|
| 分数线趋势预测 | ✅ | ✅ | 基于历史数据外推 |
| 竞争指数计算 | ✅ | ✅ | 报录比、位次分析 |
| 分数匹配推荐 | ✅ | ✅ | 规则匹配 + 排序 |
| 多校对比分析 | ✅ | ✅ | 横向对比各项指标 |
| 趋势解读报告 | ❌ | ✅ | LLM 生成文字分析 |
| 政策影响分析 | ❌ | ✅ | LLM 解读政策变化 |
| 个性化建议 | ❌ | ✅ | LLM 结合用户偏好 |

## [S6] 前端设计

**页面结构：**

```
首页（Dashboard）
├── 招生数据概览（卡片 + 图表）
├── 热门院校动态
└── 最新政策资讯

院校库
├── 院校列表（筛选 + 搜索）
├── 院校详情页（基本信息 + 专业 + 分数线）
└── 院校对比（A vs B）

数据分析（核心页面）
├── 指标选择面板（左侧）
├── 参数配置区（顶部）
├── 可视化图表区（中央）
└── 对比/叠加区（可选）

AI 助手
├── 志愿推荐（输入分数 → 推荐结果）
├── 数据解读（选择目标 → 生成报告）
└── 对话式交互（可选）

个人中心
├── 我的收藏（关注的院校）
├── 分析历史
└── 设置（AI 模式切换）
```

**UI 风格：** 深色主题 + 霓虹色点缀（金融感），ECharts 暗色主题，毛玻璃效果卡片。

## [S7] 开发顺序

1. 爬虫系统（先有数据）
2. 数据库 + 后端 API
3. 前端基础框架
4. 数据分析页面
5. AI 分析功能
6. 报名系统（最后做）
